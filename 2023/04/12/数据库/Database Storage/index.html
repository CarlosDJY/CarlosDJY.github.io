Database Storage

发表于 2023-04-12 更新于 2023-06-19 分类于 数据库

Database Storage：数据库存储是管理和组织数据的方式，通过数据库表格等结构进行。这种组织方式便于存取，支持多种操作，如查询和更新。

Database Storage

Disk Oriented DBMS

How it Works:

Disk: Directory + Files
Memory: Get request -> Directory -> get files wanted
System: Request -> get pointer to files from memory

“Pages”:

  • Hardware Page 4kB
  • OS Page 4kB
  • Database Page 512B ~ 16kB

Heap File Organization

Heap File:

  • unordered collection of pages
  • create / get / write / delete / iterate through files
  • meta data:
    keep track of pages:
    exist
    have space

Data Structure:

Link List

header for free list + header for data list
doubly linked list

Page Directory

page ID -> page
meta data: eg: free slots of page

Page:

Header:
  • page size
  • check sum
  • DBMS version
  • transaction visibility
  • compression information
Page Layout:
Tuple-oriented:

slotted pages: header with slot array
slot array map to the starting point of the tuples
keep track of

  • the # of the used slots
  • where the last slot used
    compaction: periodically remove the spaces
Log-structured:

DBMS store log records
fast write in, good for disk I/O
append log records when:

  • Insert tuples
  • delete tuples
  • update tuples
    read record:
    scan the log backward
    index for different tuples to jump through
    compaction:
  • level compaction: “merge compaction”
    bottom layer: fully compacted
  • universal compaction:
    merge adjacent files
Tuples:
Tuple Header:
  • visibility info
  • bit map for NULL values
Tuple Data:

in the order created

Denormalize Tuples:

eg: pre join
store related tuples in the same page
eg: tuple a, b + lots of tuples a, c
-> tuple a, b, c1, c2, …

Numeric Data Types:
  • fixed precision numbers
  • floats
    • IEEE-754 standard
    • stored as char array
  • large values:
    pointer to overflow storage page
    varchar data
    external value storage
    BLOB type: reference that point to external files
Record IDs:

keep track of individual tuples
record identifier:

  • page ID + offset / slot
  • (can have) file location info

System Catalogs:

internal catalogs:

  • tables, columns, indexes, views
  • users, permissions
  • internal statistics
    stores the catalog inside itself

Workloads:

On-Line Transaction Processing (OLTP):

focus more on simple writes
read / write small amount of data each time
used in online stores etc.

On-Line Analytical Processing (OLAP):

focus more on complex reads
complex reads to compute
scan a large sample
used in big data analysis etc.

Hybrid Transaction + Analytical Processing (HTAP):

OLTP + OLAP
OLTP: transactions
OLAP: analytical queries
distributed OLTP data silos + OLAP data ware house
OLTP -> extract transform load -> OLAP

Data Storage:

N-array Storage Model (NSM):

contiguous in a page
Row Store: header + data
advantages:

  • fast insert, update, delete
  • queries for entire tuples
  • ideal for OLTP
    disadvantages:
  • scan large portions of the table
  • scan for subsets
Decomposition Storage Model (DSM):

contiguous in a page
Column Store:

  • fixed-length offsets:
    same length for an attribute
  • embedded tuple IDs
    each value stored with tuple ID
    advantages:
  • reduce I/O amount
  • better query processing and compression
    disadvantages:
  • slow for inserts, updates, deletes
